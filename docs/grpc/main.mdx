<Note>
  This section documents our gRPC services based on the protobuf definitions in the
  [QFEX-org/proto](https://github.com/QFEX-org/proto) repository.
</Note>

QFEX provides 2 endpoints for gRPC:
- [mds.pfex.io](https://github.com/QFEX-org/proto/blob/main/market_data.proto) for public market and reference data.
- [trade.pfex.io](https://github.com/QFEX-org/proto/blob/main/port.proto) for order entry and private data.

These endpoints have a common set of [enums](/docs/api-reference/enums).

## Getting Started

Clone the proto definitions and generate client code for your preferred language. Example for Python:

```bash
git clone https://github.com/QFEX-org/proto.git
cd proto
pip install grpcio grpcio-tools googleapis-common-protos
python -m grpc_tools.protoc \
  -I. \
  --python_out=../client/ \
  --grpc_python_out=../client/ \
  common.proto market_data.proto port.proto
```

For Node.js projects, install:

```bash
npm install @grpc/grpc-js @grpc/proto-loader
```

## Authentication

All gRPC calls to `trade.pfex.io` require an **API Key** sent as metadata.
Read the [introduction](docs/api-reference/introduction) to learn how to get an API key.
Then add it to the `api-key` metadata field in your gRPC client.

```http
(api-key: YOUR_API_KEY_HERE)
```
